# Contributing to This Project

We welcome all contributions to this project! Whether you are a core team member or an external developer, please follow the guidelines below to ensure a smooth collaboration process.

## 1. Core Team Members (Internal Workflow)

Core project members (Rudy, Oscar, Tyler) should follow the standard **GitHub Flow** workflow directly within the main repository:

1. **Pull the latest changes:** Always start by pulling the latest `main` branch.
2. **Create a branch:** Create a new branch for your feature or bug fix (`git checkout -b feature/your-feature-name` or `fix/issue-description`). **Never commit directly to the `main` branch.**
3. **Commit your changes:** Write clear and descriptive commit messages.
4. **Push and Open a PR:** Push your branch to the remote repository and open a Pull Request (PR) against the `main` branch.
5. **Review:** Request a review from at least one other team member before merging the PR.

## 2. External Contributors (Arms-length Workflow)

If you are an external contributor, you will not have direct write access to the repository. Please use the **Fork and Pull** model:

1. **Fork the repository** to your own GitHub account.
2. **Clone** your forked repository to your local machine.
3. **Create a new branch** (`git checkout -b feature/your-feature-name`).
4. **Make your changes** and push them to your fork.
5. **Submit a Pull Request** from your fork to our `main` branch.
   We will review external pull requests within 7 days.

## 3. Reporting Bugs or Requesting Features

If you notice a bug or have a feature request, please open an issue in the GitHub issue tracker. Please include as much detail as possible (e.g., steps to reproduce, expected behavior, OS, and Docker environment).

## Code of Conduct

All contributors must abide by our [Code of Conduct](CODE_OF_CONDUCT.md).
